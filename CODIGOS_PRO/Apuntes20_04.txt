Proyecto
	Intellij: Configuraciones y dependencias en local
	Maven: manejo de librerias externas y configuraciones sobre un fichero pom.xml
	JDK: 	libreria (conjunto de elementos) que permiten la programacion Java
	JRE: 	entorno de ejecucion del codigo java
	JVM: 	maquina virtual donde se ejecuta el codigo
	IDE: 	entorno de desarrollo del programa 
	metodo main: 		el punto de entrada general de la aplicacio
	es static por: 		debe ser accesible directamente
	que quiere decir static: todo elemento definido como static, pertenece a la clase, no a la instancia del objeto

Patrones: 
	
	MVC: 	Model -> el POJO, las clases, los elementos, que van a ser trabajados en la aplicacion
			View -> la parte grafica de la aplicación -> Main -> menus
			Controller -> la parte logica de la aplicacion, la logica que se aplica en la app
			DataBase -> conexiones y configuraciones de base de datos
			Util -> para elementos miscelanea
			Exceptions -> Excepciones personalizadas
			Enviroments -> constantes o elementos que deben ser accesibles desde cualquier parte del proyecto
	Singleton	tan solo se creará una propiedad connection (concepto static)

JavaFX
	
	Stage: 	Es la ventana de la aplicacion. Se puede tener tantas como queramos
	Scene: 	Es el contenido de la ventana. Nos referimos a la parte grafica.
			Esta escena se define el fichero .fxml donde se definen cada uno de los nodos. Estos nos puedes ser: contenedor (layout) o final (los elementos que se renderizan)
			La parte logica de la escena la gestiona el controller, donde se asocia mediante un atributo llamado fx:controller y luego cada uno de los nodos fx:id
	FXMLoader: permite traer el fichero .fxml y asociarlo a la scene
	Controller: asocia la logica a un scene (fxml). Muy recomendable implementar una interfaz Initializable (se ejecuten cosas nada mas arrancar la pantalla) 
	module.info:	control de los acceso a cada parte del proyecto 
	@FXML: anotacion que permite inyeccion del elemento grafico en el controller, con la particularidad que tiene que llamarse (la variable) con el nombde del fx:id
	Acciones: se asocian listener. de forma directa: setOnXXXX(eventHandler). de forma indirecta. 					btnLogin.addEventHandler(MouseEvent.ANY, event->{});
	editCorreo.textProperty().addListener


Conceptos
	Interfaz:	define metodos sin implementar que no tienen cuerpo (abstractos). Usada para poder llevar los metodos a clases. Ademas de esto las interfaces puede tener variables (final - no pueden ser modificadas) publicas y static.
		De cuantas interfaces se puede implementar en una clase? -> de tantas como se quiera
		De cuantas clases se puede extener en una clase? -> de una
	Clase:	plantilla / molde para poder crear tantos objetos (instancias) como se quiera.
		tiene son atributos -> privados -> encapsulacion
		tiene metodos -> publicos para acceso de las instancias (getter / setter) o privados para uso del propio objeto
		tiene constructores -> si no hay nada escrito el constructor vacio. Si tengo uno escrito el vacio se enmascara

Base de datos
	Driver -> el "puente" entre el cliente y la base de datos. Es una libreria externa
	trabaja con el paquete java.sql
	url de la base de datos -> protocolo://donde_hablo/base_dato jdbc:mysql://127.0.0.1/tienda_thepw
	DAO/DTO -> ejecuciones sobre base de datos (sin logica). un dao por model. EL CRUD
	Trabajo con base de datos:
		Connection -> llegar a la base de datos
		PrepareStatement -> ejecutar query parametrizada. Parametros en base1
		Statement -> ejecutar query plana (SQL Injection)
			execute / executeUpdate
		ResultSet -> obtiene resultado de una query de seleccion
			executeQuery
			rs.next() -> pregunta si hay siguiente. En caso de haberlo se mueve el puntero para la extraccion de datos
				rs.getXXX(nombre_columna)

Excepciones
	Posibles error en codigo donde "debo" controlarlos. try-catch-finally (optativa). Las exceptiones de obligada captura son: IOException. No son necesarias las de: RuntimeExcepcion. Se puede crear personalizadas siempre y cuando extiendas una clase de Exception
		try-catch -> captura el error y le pones solucion
		throws -> avisa que el metodo puede producir un error -> el try catch será utilizado en la llamada del metodo

Arrays
	un array es una coleccion de datos, caracterizada por ser indexada (por posiciones) y tiene tamaño fijo
		int[] datos = new int[2]
		int[] datos = new int[]{4,8}
		{4,8} -> es de tipo Object[]
		int[]{4,8} -> es de tipo int[]
