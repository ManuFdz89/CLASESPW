Base de datos
				-> para poder conectar con una base de datos, necesito el Driver (traductor para entender como habla la base de datos)
				-> Connection: permite llegar a la base de datos. Es recomendable utilizar un patron Singleton. Este patron dice que: solo una instancia del tipo connection gracias al uso de las variable static
					-> Para poder iniciar la connection desde el driver, se pide la url de conexion. jdbc:mysql://127.0.0.1:3306/nombreDB
				-> DAO. Creamos las acciones CRUD de la base de datos - INSERT, UPDATE, DELETE, SELECT
					-> Statement: ejecuta query "manuales" indicando el tipo del dato en la query ('' para string). Se puede hacer un SQL Inj. execute() -> bool executeUpdate() -> int
					-> PrepareStatement: ejecuta query "parametrizadas" indicando el tipo del dato en el parametro. Mas segura que el statement. execute() executeUpdate() execute() -> bool executeUpdate() -> int
						-> los parametros utilizan una base 1-> la primera interrogacion es la 1
					-> ResultSet: obtiene resultados de select executQuery() -> resultSet
						-> para poder evaluar cada resultado rs.next() -> movia el puntero y obtenemos los datos



Clase			-> molde que representa la estructra basica de creacion de un objeto futuro. Una vez que tengo 							una clase puedo instanciar (crear) tantos objetos de esa clase como quiera. 
				-> En una clase nos entrontramos
					atributos 		-> propiedades que cualifican al objeto. deben ser privadas por concepto de ENCAPSULACION
					constructores 	-> como se crea un objeto. public NombreClase(){}. Tantos constructores como quiera. Minimo siempre el vacio
					metodos 		-> acciones, aporta funcionalidad al objeto. si marco un metodo privado solo puede ser llamado desde dentro del objeto
						getter / setter -> acceso y modificacion de variables del objeto
				-> Una variable static -> no pertenece al objeto (no necesitas una instancia del objeto), pertenece a la propia clase
				-> Una variable final aquella que puede no alterar su valor. Si la defino arriba como variable de clase, le tengo que dar el valor si o si



Exception		-> posible fallo durante la ejecucion. No es obligatorio capturarla unckecked (RuntimeExecption, NullPointerEx, ArithException, IndexOutOufBoundEx, IllegalArgumentExc). Si con obligada captura checker (IOException, FileNotFoundException). Todas las excepciones depende de Exception
				-> try (intenta ejecutar codigo) - catch (capturar el error producido) - finally (se ejecuta si o si)
				-> throws Exception. Indico en la firma del metodo que este puede provocar error y sera tratado en el sido donde se utilice



Arrays 			-> estructura de datos con principal caracteristica, no dinamica, indexada
				-> int[] datos = new int[2]
				-> int[] datos = new int[]{1,2}
				-> int[] datos = {1,2}
				-> datos[1] -> siendo la primera posicion la 0
				-> datos[1] = valor
				-> for / foreach -  el primero puede modificar el segundo solo acceder a los datos
